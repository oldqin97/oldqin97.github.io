    # git config

    1. /etc/gitconfig 文件，包含系统上每一个用户及他们仓库的通用配置

    • 如果在执行 git config 时带上 --system 选项，那么他就会读取该文件的配置变量
    • 如果它是系统配置文件，因此你需要管理员权限来修改它

    2. -/.gitconfig 只针对当前用户

    • 你可以传递 --global 选项让 Git 读写此文件，这会对你系统上的所有仓库生效
      • git config --global user.name 'xxx'
      • git config --global user.email 'xxx@'
      • 检测当前的配置信息：git config --list

    3. 当前使用仓库的 Git 目录中的 config 文件(.git/config)

    • 你可以传递 --local 选项让 Git 强制读写此文件，默认情况下就是它

    # Git alias 别名

    git 并不会在你输入部分命令时自动推断出你想要的命令

    git config --global alias.co checkout
git config --global alias.br branch
git config --global alias.ci commit
git config --global alias.st status

    # git 文件的状态划分

    对文件来划分不同的状态，以确定这个文件是否已经归于 Git 仓库来管理

    • 未跟踪：默认情况下，git 仓库下的文件也没有添加到 git 仓库管理中，我们需要 add 命令来操作
    • 已跟踪：添加到 git 仓库管理的文件处于已跟踪状态，git 可以对其进行各种跟踪管理

    已跟踪的文件又可以进行细分状态划分

    • staged: 暂存区中的文件状态
    • Unmodified: commit 命令，可以将 staged 中的文件提交到 Git 仓库
    • Modified: 修改某个文件后，会处于 Modifyed 状态

    # 检测文件的状态 git status

    查看文件的状态

    Untracked file: 未跟踪的文件, 未跟踪的文件意味着 Git 在之前的提交中没有这些文件, Git 不会自动将之纳入跟踪范围，除非你明明白白第告诉它，我需要跟踪该文件

    # 查看更加简介的状态信息
git status -s
git status --short

    # git 校验和 (commit ID)

    git 中所有的数据在存储前都会计算校验和，然后以校验和来引用

    • Git 校验和的机制是 SHA-1 散列 ( hash )
    • 40 个 16 进制字符组成的字符串，基于 git 中文件的内容或目录结构计算出来

    # 版本回退 git reset

    如果想要进行版本回退，我们需要先知道处于哪一个版本：Git 通过 HEAD 指针记录当前版本

    • HEAD 是当前分支引用的指针，它总是指向该分支上的最后一次提交
    • 理解 HEAD 的最简方式，就是将它看做该分支上的最后一次提交的快照

    通过 HEAD 来改变 Git 目前的版本指向

    1. 上一个版本就是 HEAD^, 上上个版本就是 HEAD^^
    2. 上 1000 个版本，可以使用 HEAD ～ 1000
    3. 我们可以指定某一个 commit id
    git reset --hard HEAD^

git reset --hard HEAD1000

git reset --hard 'commit id'

    # 远程仓库的验证

    • 基于 http 的凭证存储(credential storage)
      • 因为本身 http 协议是无状态的连接，所以每一个连接都需要用户名和密码，git 拥有一个凭证系统来处理这个事情
      • git config credential.helper (mac -> osxkeychain | window -> git credential manager for windows)
    • 基于 ssh 的密钥
      • secure shell 安全壳协议，以非对称加密实现身份验证，使用自动生成的公钥和私钥来简单的加密网络连接，随后使用密码认证进行登陆
      • 使用人工生成的一对公钥和私钥，通过生成的密钥进行认证，这样就可以在不输入密码的情况下登陆
      • 公钥放在待访问的电脑之中，而对于的私钥需要用户自行保管
      ssh-keygen -t ed25519 -C 'your email'
ssh-keygen -t rsa -b 1024 -C 'your email'
