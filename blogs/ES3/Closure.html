    闭包:Closure 闭包由两部分组成: 函数 和 可以访问的自由变量 一个普通函数 function, 如果它可以访问外层作用域的自由变量, 那么这个函数就是存在闭包. 广义上看,Javascript 的函数都是闭包, 从狭义角度看,JavaScript 中的一个函数,如果访问了外层作用域的变量,那么它就是一个闭包.

    function foo() {
  var name = 'foo';
  return function bar() {
    log('bar', name);
  };
}
var fn = foo();
fn(); // bar foo

    # 因为 JavaScript 的运行环境普遍存在垃圾回收器, 通过标记清除这个算法,设置一个根对象, 垃圾回收器会定期从这个根对象开始, 查找所有从根开始有引用的对象, 对于哪些没有引用到的对象, 就认为是不可达对象, 设置清除

    # 闭包导致的内存泄漏

    function foo() {
  var name = 'foo';
  function bar() {
    log(name);
  }
  return bar;
}

var fn = foo();
fn(); // 保存着foo函数内部的引用 name

fn = null; // 设置fn的函数指向null, 不再指向到foo函数内部, foo函数内部得以释放

    从 chrome 浏览器的内存观察情况上看 ,内存由低到高, 400m 为峰值, 这个时候变量 arrayFns 保存着数据的引用. 当手动将 变量指向 null 时, 存放在堆内存的数据没有引用了, 就被 chrome 浏览器释放掉 (v8) 内存断崖式下跌

    通过两个 for 循环, 先填充 100 个数据到内存中, 然后在异步队列中注册一个函数,15s 后逐步释放掉内存
